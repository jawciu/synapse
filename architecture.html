synapse

Architecture & Data Flow — A beginner's visual guide

What is this app? Synapse is a journaling app with a brain. You write (or speak) a reflection, and an AI reads it — extracting your emotional patterns, inner psychological parts, and recurring themes — then stores all of that in a database shaped like a web of connections. Over time, it builds a "graph" of who you are psychologically.

What's a "graph database"? Instead of storing data in rows like a spreadsheet, a graph database stores it as nodes (things) and edges (connections between things). Think of it like a mind map that the AI fills in automatically.

How to read these diagrams: Each section below has a plain-English explanation first, then a technical diagram. Boxes are components, arrows show data moving between them. Read top-to-bottom or left-to-right.

Section 1
The Big Picture
There are four layers in Synapse. Data flows from you (the user) → through an API → into AI agents → and finally into the database. The diagram below shows all the pieces and how they connect.
Layer 1 — You
Client (Telegram or Web Browser)
Two ways to use the app: a Telegram bot (you message it like a friend) or a web frontend (a React app in the browser). Both send your text to the same backend.
Layer 2 — API
FastAPI Server
An API is a "front door" for your backend. FastAPI listens for requests from the clients and decides what to do with them. Think of it as the receptionist that routes your message to the right department.
Layer 3 — Agents
LangGraph AI Agents
Agents are AI programs that can reason and take multiple steps to complete a task — like a human who reads something, decides what to do, looks things up, and then writes a response. LangGraph is the framework that organises those steps.
Layer 4 — Memory
SurrealDB (Graph + Vector)
The database stores everything permanently. It uses two storage modes: a graph (nodes and connections) for structured patterns, and a vector store (maths-based search) so the AI can find similar past reflections by meaning, not just keywords.

Section 2
The Reflection Pipeline — 6 Steps the AI Takes
When you submit a journal entry, it doesn't just get saved. It goes through a 6-step automated pipeline. Each step is a "node" in a LangGraph StateGraph — imagine an assembly line where each station does one job and passes the result to the next.
1
store_reflection — Your raw text is saved to the database immediately, and also converted into a "vector" (a list of numbers that captures the meaning of your text) so it can be searched later.
2
extract_patterns — A ReAct AI agent reads your reflection. Before extracting anything, it first looks up your existing patterns and reads similar past reflections — so it reuses the same labels over time instead of inventing new ones. Then it outputs structured JSON: patterns, emotions, themes, IFS parts, schemas, people, body signals.
3
update_graph — Everything extracted in step 2 gets written to the database. New patterns are created; existing ones have their occurrence count incremented. Connections (edges) are drawn between your reflection and every extracted item.
4
query_graph — The system queries your historical data: which patterns co-occur most, what triggers negative emotions, which patterns are most central in your graph. This context is gathered to feed into the next steps.
5
generate_insights — GPT-4o synthesises your current reflection + the extracted items + the historical graph data into a short, warm insight paragraph — something a therapist might observe.
6
generate_followups — GPT-4o generates exactly 3 follow-up questions tailored to what it found, to help you go deeper.

What is a StateGraph? LangGraph's StateGraph is like a flowchart where each box (node) is a Python function. Each function receives the current "state" (think of it as a shared notepad), does its job, and updates the notepad. The next function picks up where the last one left off. All 6 nodes share the same state object — so step 5 can see everything that steps 1–4 added.
Section 3
What is a ReAct Agent? The Two Agents Explained
A ReAct agent is an AI that alternates between Reasoning (thinking about what to do) and Acting (calling a tool to get information). It keeps looping — think, act, observe result, think again — until it has enough to give a final answer. Synapse uses two different ReAct agents for two different purposes.
Agent 1 — Inside the Pipeline
Extraction Agent
Its only job is to read one reflection and output a JSON object of everything psychological it can find.

Before extracting, it calls two tools: one to see what patterns already exist in your graph (so it reuses "perfectionism" instead of creating "being too perfect"), and one to retrieve similar past reflections for context. This is called RAG — Retrieval-Augmented Generation. The AI's output is grounded in real data, not just guesses.

Agent 2 — The Ask Tab
Chat Agent (13 tools)
This agent answers your freeform questions about your own graph — like "what patterns do I repeat?" or "what triggers my anxiety?"

It has 13 tools, each of which runs a different database query. The agent decides which tools to call based on your question. It also has memory via thread IDs — it remembers what you said earlier in the same conversation.

Tools vs. direct calls: A "tool" in AI is just a Python function the model is allowed to call. The model decides when and whether to call it — you don't hardcode the logic. This makes agents flexible: they can handle many types of questions without needing a different code path for each one.
Section 4
The Database — Your Psychological Graph
SurrealDB stores your data as a graph: nodes are "things" (a pattern, an emotion, a person), and edges are "relationships between things" (this reflection reveals this pattern; this IFS part protects against this schema). Over time, the graph fills in and the AI can traverse connections to find deeper roots.
Concept
Graph nodes = psychological concepts
Each unique concept (e.g. "perfectionism", "anxiety", "inner critic") is stored once as a node. Every time it appears in a new reflection, its occurrence count goes up. You can see what you think about most often.
Concept
Graph edges = connections
An edge says "this reflection REVEALS this pattern" or "this person TRIGGERS this pattern". The graph lets the AI traverse: "what emotions are triggered by what themes?" without scanning all your text every time.
Concept
Vector embeddings = search by meaning
Some nodes also have an "embedding" — a list of ~1500 numbers that encodes what the text means mathematically. Two concepts with similar meanings will have similar numbers. This lets the AI search by meaning ("find patterns similar to perfectionism") not just by exact keyword match.

Section 5
Voice Notes — How Speech Becomes Graph Data
When you send a voice message in Telegram, it goes through an extra first step: speech-to-text transcription via OpenAI's Whisper model. After that, the transcript is treated exactly like typed text and runs through the full 6-step pipeline. The diagram below traces every step from your mouth to the database.
Step A
Download the audio file
Telegram stores voice notes on its servers. The bot downloads it as a temporary .ogg file (a compressed audio format) to the local machine. This file is only kept for a few seconds.
Step B
Transcribe with Whisper
The audio file is sent to OpenAI's Whisper-1 model — a specialised AI trained only on speech recognition. It returns a plain text string. The temp file is then immediately deleted. Whisper supports 57 languages automatically.
Step C
Full pipeline runs on transcript
The transcript text is passed into the same 6-node LangGraph pipeline as any typed reflection. From this point, voice and text are treated identically — both end up as nodes and edges in SurrealDB.

Why delete the temp file? Voice notes contain personal information. The bot never stores audio on disk longer than the few seconds it takes to transcribe. After Whisper returns the text, the audio file is deleted in a finally block — which means it gets deleted even if the transcription fails partway through.
Section 6
Glossary — Key Terms
Term
LLM (Large Language Model)
The core AI technology. GPT-4o is an LLM — a model trained on vast amounts of text that can read, reason, and generate language. In this app it does extraction, insight generation, and chat responses.
Term
Agent
An AI program that can take multiple steps, use tools, and make decisions autonomously. Unlike a single LLM call (input → output), an agent can loop: think → act → observe → think again.
Term
RAG (Retrieval-Augmented Generation)
Before generating an answer, the AI first retrieves relevant information from a database. This grounds its output in real data rather than relying solely on what it learned during training.
Term
Vector / Embedding
A way to represent text as a list of numbers. Texts with similar meanings produce similar number lists. This lets you search by meaning: "find reflections similar to this one" without needing exact keyword matches.
Term
Graph Database
A database where data is stored as nodes (things) and edges (connections). Great for relationship-heavy data like "this pattern co-occurs with that pattern 14 times" — queries that would be slow in a regular SQL database.
Term
IFS / Schema / CBT
Therapy frameworks the app borrows language from. IFS (Internal Family Systems) talks about "parts" of the psyche. Schema Therapy identifies deep patterns from childhood. CBT (Cognitive Behavioural Therapy) focuses on thought patterns. The app uses these as labelling systems, not as diagnosis.
